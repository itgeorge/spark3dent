\# Draft



Invoice correction currently only supports correcting invoices as long as the number and date changes don't conflict with other existing invoices.

We should add the capability to correct a sequence of invoices, e.g. if a number or date was wrong and that conflicts with future invoices, the tool should allow automatically correcting all subsequent invoices after the invoice we're correcting.



Expand this doc with a detailed plan on achieving this in a TDD way.
